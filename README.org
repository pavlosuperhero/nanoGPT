* Architecture Overview
This repository tracks the implementation of a micro-scale Generative Pre-trained Transformer (GPT). The system is designed as a spatial pipeline that digitizes discrete text, projects it into a continuous high-dimensional manifold, mixes contextual information via self-attention, and decodes the result back into vocabulary probabilities.

* Phase 1: Tokenization & Vocabulary Mapping
Before computation begins, raw text must be digitized. Tokens are discrete indices mapped to a vocabulary space. They do not carry semantic meaning at this stage; they are strictly spatial pointers.

** Implementation
#+begin_src python :results output :session *nanoGPT*
with open('input.txt', 'r', encoding='utf-8') as f:
    text = f.read()

chars = sorted(list(set(text))) 
vocab_size = len(chars)

# Spatial lookup tables for encoding/decoding
stoi = { ch:i for i,ch in enumerate(chars) }
itos = { i:ch for i,ch in enumerate(chars) }

encode = lambda s: [stoi[c] for c in s] 
decode = lambda l: ''.join([itos[i] for i in l]) 
#+end_src

** Scientific Context
Tokenization is a lossless mapping from a string sequence to an integer space $\mathbb{Z}^{+}$. Advanced production models use Byte Pair Encoding (BPE), a data compression technique that iteratively merges the most frequent pairs of bytes to optimize context window usage.
- *Documentation*: [[https://github.com/openai/tiktoken][Tiktoken BPE Algorithm]]

* Phase 2: Embedding & Spatial Projection
Discrete indices lack physical relationship or distance metrics. We project them into an $N$-dimensional continuous space where mathematical distance equals semantic proximity.

** Implementation
#+begin_src python :results output :session *nanoGPT-Training*
import torch
import torch.nn as nn

# Context length (T) and Batch size (B)
block_size = 8 
batch_size = 4 

# Projection matrices
token_embedding_table = nn.Embedding(vocab_size, n_embd)
position_embedding_table = nn.Embedding(block_size, n_embd)
#+end_src

** Scientific Context & Mathematics
The =nn.Embedding= layer acts as a physical lookup table. Mathematically, it performs a spatial projection by multiplying a one-hot encoded vector $x \in \{0,1\}^{|V|}$ by a learnable weight matrix $W \in \mathbb{R}^{|V| \times d_{model}}$.

To prevent the model from becoming spatially blind to sequence order, a positional vector is added to the token vector. The resulting spatial object is physically deformed by its timestamp:
$$X_{final} = \text{Embedding}(Token) + \text{Embedding}(Position)$$

- *Documentation*: [[https://pytorch.org/docs/stable/generated/torch.nn.Embedding.html][torch.nn.Embedding]]

* Phase 3: The Self-Attention Mechanism (Contextual Mixing)
Tokens in a sequence are initially isolated. Self-attention acts as a dynamic routing protocol, allowing past tokens to physically bleed their geometric features into future tokens to build contextual awareness.

** Implementation (Mathematical Trick)
#+begin_src python :results output :session *nanoGPT-Training*
from torch.nn import functional as F

B, T, C = 4, 8, 32
x = torch.randn(B, T, C)

# Lower triangular matrix acts as a causal barrier
tril = torch.tril(torch.ones(T, T))
weights = torch.zeros((T, T))

# Apply the spatial mask (futures become -inf)
weights = weights.masked_fill(tril == 0, float('-inf'))

# Normalize into percentages
weights = F.softmax(weights, dim=-1)

# Execute spatial mixing
xbow = weights @ x
#+end_src

** Scientific Context & Mathematics
This is the core of the Transformer engine. 
1. *Causal Masking*: The upper triangle of the matrix is replaced with $-\infty$. 
2. *Softmax*: Transforms raw spatial distances into a valid probability distribution. Since $e^{-\infty} = 0$, the future is hardware-blocked, preventing data leakage during training.
$$ \text{Softmax}(z_i) = \frac{e^{z_i}}{\sum_{j} e^{z_j}} $$
3. *Matrix Multiplication (Dot Product)*: Computes the spatial alignment between states. 

[Image of vector dot product geometric interpretation]

$$ \text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}} + M\right)V $$

- *Documentation*: [[https://pytorch.org/docs/stable/generated/torch.nn.functional.softmax.html][torch.nn.functional.softmax]]
- *Paper*: [[https://arxiv.org/abs/1706.03762][Attention Is All You Need (Vaswani et al., 2017)]]

* Phase 4: Linear Decoding & Autoregressive Generation
The fully contextualized high-dimensional tensor is routed through a final projection layer (=lm_head=) to decode it back into a flat vocabulary space.

** Implementation
#+begin_src python :results output :session *nanoGPT-Training*
# Decoding the manifold back to vocabulary dimensions
logits = self.lm_head(x) 

# Focus on the temporal edge (the last state)
logits = logits[:, -1, :] 
probs = F.softmax(logits, dim=-1)

# Sampling based on calculated physical proximity
idx_next = torch.multinomial(probs, num_samples=1)
#+end_src

** Scientific Context
The linear layer evaluates the scalar product between the incoming contextualized vector and the frozen weights of the vocabulary. The resulting logits represent the geometric proximity of the context to every possible next token. These logits are compressed by Softmax, creating a categorical probability distribution that is sampled by a multinomial generator to predict the next token.
- *Documentation*: [[https://pytorch.org/docs/stable/generated/torch.multinomial.html][torch.multinomial]]
